TicketerApp - Event ticketing flows

Context

Mobile-first ticketing concept for concerts and local events. Target: quick purchase in under 2 minutes. Platform: responsive web + iOS.

Problem

Users abandon during seat selection and payment. We aimed to reduce decision friction and improve trust (fees visibility, clear steps).

Process

  • Competitive scan of 3 ticketing sites; mapped steps & pain points.
  • Flows for discovery -> event -> seating -> extras -> payment.
  • Wireframes to Hi-fi in Figma; prototyped the 3 key tasks.
  • Patterns: progress tracker, fee disclosure, seat map zoom, saved payments.

Solution

Annotated screens for search, event detail, seat map, cart, checkout. Microcopy for fees and error states. Edge cases (sold out, partial seats).
